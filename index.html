Over mij

Welkom op mijn portfolio. Ik ben Agit Basar. Een student in de opleiding Toegepaste Informatica aan het Thomas More in Geel. Ik zit momenteel in mijn derde jaar. Als specialisatie heb ik gekozen voor applicatieontwikkeling.

Mijn interesse in applicaties ontwikkelen ontstond tijdens mijn studies in secundair onderwijs. In het middelbaar heb ik Boekhouden-Informatica gevolgd. Ik had als eerst voor dit studie gekozen wegens de interesse in boekhouden. Snel veranderde deze interesse naar programmeren na het maken van mijn eerste programma op school.

In dit portfolio documenteer ik de vorderingen die ik in de IT Factory van Thomas More maak. In dit porfolio kunt u mijn projecten bekijken en ook meer informatie vinden over mijn stage dat ik heb gelopen in de derde jaar Toegepaste Informatica.

Projecten

Application Challenge - Angular

Tijdens mijn eerste semester in het derde jaar heb ik deelgenomen aan een Application Challenge. De uitdaging was om een applicatie in één week tijd te programmeren in Angular met een groep van vier studenten. De applicatie is een soort van matchmaking systeem. Waar er twee soorten gebruikers kunnen zijn. De ‘Makers’ en het bedrijven. ‘Makers’ zijn studenten, stagiairs of mensen die in hun vrije tijd met IT bezig zijn. In dit applicatie kunnen bedrijven korte opdrachten zetten die niet zo dringend zijn, maar die ze wel graag willen laten uitvoeren of die ze willen onderzoeken. De ‘Makers’ kunnen deze opdrachten bekijken en hiervoor inschrijven. Nadat er een paar inschrijvingen zijn kan het bedrijf een ‘Maker’ kiezen en hiermee in contact komen.

Spring Boot applicatie

Voor het vak Java Advanced Topics heb ik samen met mijn team een Applicatie geprogrammeerd. Het doel van de opdracht was om een applicatie te maken door gebruik te maken van microservices. Bij een traditionele "Monolitische applicatie" kan het voorvallen dat er meer vraag is naar één bepaalde service. Als dit gebeurt dan kunnen er lange wachttijden onstaan voor de gebruikers. Als er meer vraag is naar 1 bepaalde service, dan kunnen daar meerdere instanties van gestart worden om de vraag er naar te beantwoorden bij een microservice. Op deze manier wordt onze applicatie zeer dynamisch naargelang gebruik ervan. Onze applicatie heet Dreamteam. In dit applicatie wordt er gefocust rond voetbal. Het doel is om van verscheidene competities de teams en spelers te tonen. Hiermee zal de gebruiker dan de mogelijkheid hebben. om zijn eigen Dreamteam samen te stellen met zijn favoriete spelers.

Project 4.0

Tijdens mijn derde jaar ben ik samen met mijn team bezig geweest aan project 4.0. In dit project hebben we een opdracht gekregen van het IT-bedrijf Ometa om een mobiele applicatie te programmeren voor het bedrijf Duracell. Duracell wilt graag hun productie beter monitoren. Wanneer er een probleem voordoet op de werkvloer zoals een machine dat vast zit of als er een machine een te hoge temperatuur heeft moet dit zo snel worden opgelost. Soms kan dit even duren waardoor de productie zelfs stil kan gezet worden. Om deze problemen zo snel mogelijk op te lossen kunnen we op voorhand thresholds instellen op de sensoren in de werkvloer. Wanneer de threshold wordt overschreden krijgt de mobiele applicatie via de Ometa-framework een melding van de sensor. In de applicatie staan ook alle vorige meldingen waar de gebruiker deze kan bekijken, bewerken en verwijderen. De gebruiker kan ook een weergave van de sensoren bekijken in een grafiek. Zo kunnen de werknemers op voorhand voorbereiden voor het probleem of zelfs het probleem vermijden.

Stage

Angular developer bij E.care
02/2020 - 05/2020

Tijdens het laatste jaar van mijn studies in de IT Factory van Thomas More liep ik gedurende 13 weken stage bij het bedrijf E.care. E.care is een bedrijf dat software maakt, implementeert en verkoopt aan ziekenhuizen, meer specifiek de spoedafdeling. De bedrijf is nog niet zolang geleden overgenomen door een Duitse bedrijf namelijk genaamd Meona. De bedrijf E.care is gelegen in Turnhout. De bedrijf bestaat uit 20-aantal werknemers.

Als taak heb ik gekregen om de E.care software om te zetten naar Angular. Momenteel is de E.care software ouderwets. Men wil graag de software ook sneller laten werken. Het is cruciaal in omgeving zoals in een ziekenhuis om een snel werkende en gebruiksvriendelijke software te hebben. Door de oude software om te zetten naar Angular kunnen we hier op focussen. Wanneer een nieuwe versie wordt uitgebracht moet elke ziekenhuis hun software updaten. Hier willen ze er ook van af zijn. Want bij elke update moet de software herstart worden en dit zijn kostbare tijden. Dit kunnen we oplossen door oude software op een webapplicatie te zetten. Zo hoeven we alleen maar de server up te daten. Zo hoeven we alleen maar de server up te daten indien het nodig is.
